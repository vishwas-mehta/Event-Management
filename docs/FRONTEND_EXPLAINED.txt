PART 1: PROJECT SETUP - FROM SCRATCH
================================================================================

1.1 CREATING THE FRONTEND FOLDER
---------------------------------
This project uses Vite (a fast build tool) with React and TypeScript.

To create a similar frontend from scratch, you would run:

    npm create vite@latest frontend -- --template react-ts
    cd frontend 

This creates a new Vite project with React and TypeScript template.


1.2 INSTALLING ALL DEPENDENCIES
--------------------------------
After creating the project, install all required packages:

Core Dependencies (production):
    npm install react react-dom                    # Core React library
    npm install react-router-dom                   # For navigation/routing
    npm install axios                              # For HTTP API calls
    npm install bootstrap react-bootstrap          # UI component library
    npm install bootstrap-icons                    # Icon library
    npm install date-fns                           # Date formatting utilities
    npm install react-hook-form                    # Form handling
    npm install lucide-react                       # Additional icons

Development Dependencies (already included by Vite):
    npm install -D typescript @types/react @types/react-dom
    npm install -D vite @vitejs/plugin-react
    npm install -D eslint @eslint/js typescript-eslint

Full npm install command to replicate this project:
    npm install axios bootstrap bootstrap-icons date-fns lucide-react react react-dom react-bootstrap react-hook-form react-router-dom


1.3 CONFIGURATION FILES
------------------------
vite.config.ts - Build tool configuration:
    - Sets development server to port 3000
    - Proxies /api requests to backend at localhost:5000
    - Configures path alias @ for src directory

.env file:
    VITE_API_URL=http://localhost:5000/api  (or leave empty to use default)

2.1 THE RENDERING CHAIN (Step by Step)
---------------------------------------

STEP 1: index.html (Entry HTML file)
    Location: /frontend/index.html
    
    This is the ONLY HTML file. It contains:
    - A single <div id="root"></div> where React mounts
    - A script tag loading src/main.tsx
    
    Flow: Browser loads index.html â†’ loads main.tsx â†’ React takes over


STEP 2: main.tsx (JavaScript Entry Point)
    Location: /frontend/src/main.tsx
    
    What it does:
    1. Imports React and ReactDOM
    2. Imports CSS: Bootstrap CSS, custom index.css, Bootstrap icons
    3. Finds the #root div in index.html
    4. Renders the <App /> component inside React.StrictMode
    
    Code explanation:
        ReactDOM.createRoot(document.getElementById('root')!).render(
            <React.StrictMode>
                <App />           â† This is where everything starts
            </React.StrictMode>,
        )


STEP 3: App.tsx (Main Application Component)
    Location: /frontend/src/App.tsx
    
    Structure:
        <BrowserRouter>            â† Enables URL-based navigation
            <AuthProvider>         â† Provides authentication state to ALL components
                <AppRoutes />      â† Defines which page shows for which URL
                <Chatbot />        â† Floating chatbot widget (always visible)
            </AuthProvider>
        </BrowserRouter>
    
    Key concept: Everything is wrapped in AuthProvider, so any component
    in the app can know if user is logged in, who they are, and their role.


2.2 THE ROUTING SYSTEM (What Page Shows When)
----------------------------------------------

App.tsx defines ALL routes using React Router's <Routes> and <Route>:

PUBLIC ROUTES (anyone can access):

    /events         â†’ EventsPage       (browse all events)
    /events/:id     â†’ EventDetailsPage (single event details)

ADMIN ROUTES (role=admin only):
    /admin/dashboard           â†’ AdminDashboard
    /admin/users              â†’ ManageUsersPage
    /admin/organizers/pending â†’ PendingOrganizersPage
    /admin/reports            â†’ ReportedEventsPage

ORGANIZER ROUTES (role=organizer only):
    /organizer/dashboard       â†’ OrganizerDashboard
    /organizer/events/create   â†’ CreateEventPage
    /organizer/events/:id/edit â†’ EditEventPage
    /organizer/events/:id/manage â†’ ManageEventPage

ATTENDEE ROUTES (role=attendee only):
    /attendee/dashboard â†’ AttendeeDashboard
    /attendee/bookings  â†’ MyBookingsPage

SHARED PROTECTED ROUTE:
    /profile â†’ ProfilePage (any logged-in user)

CATCH-ALL:
    /* â†’ Redirects to home page


2.3 HOW PROTECTED ROUTES WORK
------------------------------

Every protected route is wrapped like this:
    <MainLayout>
        <ProtectedRoute allowedRoles={[UserRole.ATTENDEE]}>
            <AttendeeDashboard />
        </ProtectedRoute>
    </MainLayout>

ProtectedRoute component (src/components/Common/ProtectedRoute.tsx):
    1. Shows loading spinner while authentication is being checked
    2. If no user logged in â†’ redirects to /login
    3. If user status is not 'active' â†’ shows "Pending Approval" message
    4. If user role not in allowedRoles â†’ shows "Access Denied"
    5. Otherwise â†’ renders the child component

MainLayout component (src/components/Layout/MainLayout.tsx):
    Wraps every page with:
        <div>
            <Navbar />    â† Top navigation bar
            <main>        â† Page content goes here
                {children}
            </main>
            <Footer />    â† Bottom footer
        </div>

PART 4: REUSABLE COMPONENTS EXPLAINED
================================================================================

4.1 LAYOUT COMPONENTS
----------------------

MainLayout.tsx - Page wrapper
    Purpose: Wraps every page to show consistent Navbar and Footer
    Usage: <MainLayout><SomePage /></MainLayout>
    How it works: Uses {children} prop to render any page content between 
    Navbar and Footer

Navbar.tsx - Top navigation
    Purpose: Shows navigation links based on user role
    Features:
    - Always shows: "EventHub" brand, "Browse Events"
    - If logged out: shows "Login" and "Sign Up" buttons
    - If logged in: shows Dashboard link, user dropdown with profile/logout
    - Attendee sees: "My Bookings" link
    - Organizer sees: "Create Event" link
    - Admin sees: "Pending Approvals" link
    How role checking works: Uses useAuth() hook to get user, then checks 
    user.role to conditionally render links

Footer.tsx - Bottom footer
    Purpose: Shows copyright and links
    Static: Same for all users


4.2 UTILITY COMPONENTS
-----------------------

LoadingSpinner.tsx
    Purpose: Shows spinning animation while data loads
    Props: fullScreen (boolean) - if true, centers spinner on full page
    Usage: {loading && <LoadingSpinner />}

ErrorAlert.tsx
    Purpose: Shows error message in Bootstrap alert
    Props: message (string), onClose (function)
    Usage: {error && <ErrorAlert message={error} onClose={() => setError('')} />}

Pagination.tsx
    Purpose: Shows page numbers for navigating lists
    Props: currentPage, totalPages, onPageChange
    Usage: <Pagination currentPage={1} totalPages={5} onPageChange={handlePage} />

ProtectedRoute.tsx
    Purpose: Guards routes from unauthorized access
    Props: children, allowedRoles (array), requireActive
    Logic: Checks auth state, redirects if needed, renders children if OK


4.3 EVENT COMPONENTS
---------------------

EventCard.tsx
    Purpose: Displays event summary in a card format (used in listings)
    Props: event (EventType object)
    Shows: Banner image, category badge, price, title, date, location, rating
    Features:
    - Shows "SOLD OUT" badge if no tickets available
    - Shows "Free" badge if price is 0
    - Links to /events/${event.id} on click


4.4 PAYMENT COMPONENTS
-----------------------

PaymentModal.tsx
    Purpose: Shows Razorpay-style payment modal for paid tickets
    Props: show, onHide, ticketName, ticketPrice, quantity, eventName, 
           onPaymentComplete, loading
    Features:
    - Shows order summary with ticket price, quantity, convenience fee, total
    - Dummy payment options (UPI, Card, Net Banking, Wallets)
    - Calls onPaymentComplete when user "pays"

5.1 OPENING THE APPLICATION (What Renders First)
================================================================================

When user opens http://localhost:3000:

STEP 1: Browser loads index.html
    - Renders empty <div id="root"></div>
    - Loads main.tsx script

STEP 2: main.tsx executes
    - Imports bootstrap and custom CSS (styles load)
    - Calls ReactDOM.createRoot() on #root element
    - Renders <App /> component

STEP 3: App component renders
    Structure rendered:
        <BrowserRouter>
            <AuthProvider>           â† This runs first
                <AppRoutes />        â† Then this
                <Chatbot />
            </AuthProvider>
        </BrowserRouter>

STEP 4: AuthProvider initializes (from AuthContext.tsx)
    What happens:
    1. Creates state: user=null, token=localStorage.getItem('token'), loading=true
    2. useEffect triggers immediately (runs loadUser function)
    3. If token exists in localStorage:
       - Calls authApi.getCurrentUser() â†’ makes HTTP GET to /api/auth/me
       - If successful: sets user state with returned user data
       - If fails: removes invalid token from localStorage
    4. Sets loading=false

STEP 5: AppRoutes component renders
    - URL is "/" so matches: <Route path="/" element={<MainLayout><HomePage /></MainLayout>} />
    - MainLayout renders Navbar + HomePage + Footer

STEP 6: HomePage renders
    - Shows hero section with "Discover & Book Amazing Events"
    - Shows feature cards and "How it works" section

WHAT USER SEES:

    - Middle: Beautiful landing page with hero image and text

5.2 REGISTRATION FLOW (Creating an Attendee Account)
================================================================================

User clicks "Sign Up" button â†’ Browser navigates to /register

STEP 1: Route matches
    <Route path="/register" element={<MainLayout><RegisterPage /></MainLayout>} />

STEP 2: RegisterPage component renders (from pages/Auth/RegisterPage.tsx)
    - Creates form state with useState:
        formData = { email, password, confirmPassword, firstName, lastName, 
                     phoneNumber, role: 'attendee' }
        error = ''
        success = ''
        loading = false

STEP 3: User fills form and selects "Attend events" option (role='attendee')

STEP 4: User clicks "Create account" button â†’ handleSubmit runs:
    
    a) Form validation:
        - Check if passwords match
        - Check password length >= 6
        
    b) If validation passes:
        setLoading(true);
        await register({...formData});  â† Uses useAuth() hook's register function
        navigate('/');  â† Redirects to home page

STEP 5: register() function in AuthContext.tsx:
    
    const register = async (data: RegisterData) => {
        const response = await authApi.register(data);  â† API call
        const { user: userData, token: userToken } = response.data;
        
        // For attendee (status='active'), auto-login:
        localStorage.setItem('token', userToken);
        setToken(userToken);
        setUser(userData);
    };

STEP 6: authApi.register() in api/auth.api.ts:
    
    register: async (data: RegisterData) => {
        const response = await api.post('/auth/register', data);
        return response.data;
    }

STEP 7: api.post() in api/axios.ts:
    - Uses configured axios instance
    - BaseURL: http://localhost:5000/api
    - Makes POST to http://localhost:5000/api/auth/register
    - Body: { email, password, firstName, lastName, role: 'attendee', phoneNumber }

STEP 8: Backend processes request and returns:
    {
        success: true,
        message: "Registration successful",
        data: {
            user: { id, email, firstName, lastName, role: 'attendee', status: 'active' },
            token: "jwt.token.here"
        }
    }

STEP 9: Frontend stores token and user:
    - localStorage.setItem('token', 'jwt.token.here')
    - setUser(userData) â†’ triggers re-render
    - navigate('/') â†’ redirects to homepage

STEP 10: App re-renders with authenticated user
    - AuthContext now has user object
    - Navbar re-renders showing user name dropdown instead of Login/Sign Up
    - User sees: Dashboard link, My Bookings link, user dropdown

5.3 LOGIN FLOW (Existing Attendee Logging In)
================================================================================

User clicks "Login" in navbar â†’ Browser navigates to /login

STEP 1: LoginPage renders (from pages/Auth/LoginPage.tsx)
    - Creates state: email='', password='', error='', loading=false
    - Gets login function from useAuth() hook

        setToken(userToken);
        setUser(userData);
    };

    POST /api/auth/login
    Body: { email: "user@example.com", password: "secret123" }

    {

5.4 BROWSING EVENTS (EventsPage Flow)

User clicks "Browse Events" in navbar â†’ navigates to /events

STEP 2: useEffect hooks trigger on mount:

    const response = await eventsApi.getEvents(filters);
    setEvents(response.data.events);

    setLoading(false);

STEP 5: eventsApi.getEvents() in api/events.api.ts:
    
    getEvents: async (filters) => {
        const params = new URLSearchParams();
        Object.entries(filters).forEach(([key, value]) => {
            if (value) params.append(key, value.toString());
        });
        const response = await api.get(`/events?${params.toString()}`);
        return response.data;
    }
    
    API call: GET /api/events?page=1&limit=12&sortBy=date&order=ASC

STEP 6: Backend returns:

        success: true,
        data: {

5.9 HOW THE API LAYER WORKS

All API calls go through: src/api/axios.ts

AXIOS INSTANCE CONFIGURATION:
    const API_URL = import.meta.env.VITE_API_URL || 'http://localhost:5000/api';
    
    const api = axios.create({
        baseURL: API_URL,
        headers: { 'Content-Type': 'application/json' }
    });

REQUEST INTERCEPTOR (runs before every request):
    api.interceptors.request.use((config) => {
        const token = localStorage.getItem('token');
        if (token) {
            config.headers.Authorization = `Bearer ${token}`;
        }
        return config;
    });
    
    This automatically adds the JWT token to every API call.

RESPONSE INTERCEPTOR (runs for every response):
    api.interceptors.response.use(
        (response) => response,  â† Success: just return response
        (error) => {
            // Handle 401 errors (unauthorized)
            if (error.response?.status === 401 && hasStoredToken && !isAuthEndpoint) {
                localStorage.removeItem('token');
                window.location.href = '/login';
            }
            return Promise.reject(error);
        }
    );
    
    If backend returns 401 (token expired), auto-logout and redirect to login.

SUMMARY: KEY CONCEPTS
================================================================================

1. COMPONENT ARCHITECTURE:
   - Pages are in /pages - full page views
   - Components are in /components - reusable pieces
   - APIs are in /api - backend communication
   - State is in /contexts - global app state

2. DATA FLOW:
   User Action â†’ Event Handler â†’ API Function â†’ Backend â†’ Response â†’ State Update â†’ Re-render

3. AUTHENTICATION:
   - AuthContext provides user state to entire app
   - Token stored in localStorage for persistence
   - axios interceptor adds token to all requests
   - ProtectedRoute component guards pages

4. ROUTING:
   - React Router handles URL-to-component mapping
   - useNavigate() for programmatic navigation
   - useParams() for getting URL parameters

5. STATE MANAGEMENT:
   - useState for component-local state
   - useContext for global state (auth)
   - useEffect for side effects and data fetching
